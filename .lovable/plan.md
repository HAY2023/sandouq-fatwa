

# خطة: إصلاح مشكلة تسجيل الدخول + إشعارات في الوقت الحقيقي مع عداد مباشر

## المشكلة الأولى: لا يمكن الدخول إلى الإعدادات

**السبب**: حسابك مقفل بسبب 5 محاولات دخول فاشلة. القفل ينتهي تلقائياً بعد 15 دقيقة، لكن إذا حاولت أثناء القفل يُعاد تعيين المؤقت.

**الحل**: 
- إعادة تعيين عداد المحاولات الفاشلة وإزالة القفل من قاعدة البيانات فوراً
- إضافة رسالة خطأ واضحة تخبرك عندما يكون الحساب مقفلاً وكم تبقى من الوقت (بدلاً من "كلمة المرور غير صحيحة")

## المهمة الثانية: إشعارات في الوقت الحقيقي مع عداد مباشر

الإشعارات في الوقت الحقيقي موجودة بالفعل في لوحة الإدارة (صوت + إشعار متصفح عند وصول سؤال جديد). سيتم تحسينها:

1. **عداد مباشر في الهيدر** - إضافة شارة (badge) تعرض عدد الأسئلة الجديدة منذ فتح الصفحة، تتحدث تلقائياً
2. **رسالة منبثقة محسّنة** - عرض نص السؤال وفئته في الإشعار المنبثق
3. **تحديث تلقائي للعداد الرئيسي** - العداد الكبير في أعلى الصفحة يتحدث فوراً عند وصول سؤال

---

## التفاصيل التقنية

### 1. إصلاح القفل (Migration SQL)
```sql
UPDATE admin_credentials SET failed_attempts = 0, locked_until = NULL;
```

### 2. تحسين رسائل الخطأ في `handleLogin` (`Admin.tsx`)
- استخدام RPC function جديدة `check_admin_lock_status` تُرجع حالة القفل والوقت المتبقي
- أو تعديل `verify_admin_password` لإرجاع رسالة خطأ مفصلة بدلاً من `false` فقط
- عرض رسالة "الحساب مقفل، حاول بعد X دقائق" بدلاً من "كلمة المرور غير صحيحة"

### 3. عداد الأسئلة الجديدة في الوقت الحقيقي (`Admin.tsx`)
- إضافة state `newQuestionsCount` يزداد مع كل سؤال جديد عبر Realtime
- عرض شارة حمراء على أيقونة الجرس في الهيدر
- إضافة animation (pulse) للعداد عند وصول سؤال جديد

### الملفات المتأثرة:
- `src/pages/Admin.tsx` - تحسين handleLogin + عداد مباشر
- Migration SQL جديد - إعادة تعيين القفل + إضافة function جديدة
